# FW Visual Flow Studio — обзор и интро

## Интро: что это за концепт
FW Visual Flow Studio — концептуальная визуальная среда, где интерфейсы, сценарии, данные и тесты
собираются в единой канве. Идея появилась более 10 лет назад, но сегодня её поддерживают современные
технологии: реактивные модели данных, AI-ассистенты, мультиплатформенные рантаймы и автоматизированные
тесты.

**В одном предложении:** мы собираем экран мышью, фиксируем сценарий, сразу получаем автотест и
публикуем результат в веб, десктоп и терминал.

## Proof of Concept: готовность и проверка
- ✅ **UI прототип**: лендинг и компоненты уже доступны в проекте.
- ✅ **Склейка модулей**: присутствует базовый слой «клея» между компонентами, сценариями и данными.
- ✅ **Тестируемость**: есть каркас автотестов компонентов и сценарная логика для записи переходов.

## Что уже есть (MVP-профиль)
- Визуальный редактор (канвас, инспектор, слои).
- Сценарный движок (переходы, условия, фиксация действий).
- Каталог компонентов (метаданные, версии, совместимость).
- Базовые шаблоны экранов и сценариев.
- Пайплайн публикации в web/desktop/terminal.

## Быстрый старт
```bash
npm install
npm run dev
```

Откройте `http://localhost:5173`, чтобы увидеть UI-прототип.

## Как настраивать
На текущем этапе конфигурация минимальна и хранится в коде. Для масштабирования предлагается:
- Перенести источники данных и окружения в `.env`.
- Вынести конфиги сценариев в JSON/SDK пакет.
- Добавить централизованный каталог компонентов (registry).

## Как тестировать
### Компоненты (UI)
```bash
npm run test
```

### Проверка сценариев (ручной PoC-подход)
1. Запустите интерфейс: `npm run dev`.
2. Пройдите сценарий на UI (например, «Оплата телефона»).
3. Зафиксируйте ожидаемые переходы и события в форме сценария.

## Слой «клея» (Glue)
«Клей» — это связка между:
- **компонентами** (props, events, methods),
- **сценариями** (узлы, переходы, условия),
- **данными** (state, источники, реактивные подписки).

Его задача — гарантировать, что при смене платформы меняется только адаптер рендера, а логика остаётся
единой.

## Предлагаемое улучшение Fitch (качественное исследование)
### Исследование: что изменилось за 10 лет
- **AI-ассистенты** ускоряют сбор сценариев и автотестов.
- **Web-компиляция** (Vite, SWC, esbuild) ускорила сборки.
- **Мультиплатформенные UI** (Web, Desktop, Terminal) стали устойчивым трендом.
- **Наблюдаемость** (логирование, трейсинг) стала стандартом качества.
- **Интеграции** (API Gateway, GraphQL, gRPC) упростили интеграцию с бэкендом.

### Fitch-улучшение (идея)
Ниже — список предлагаемых улучшений. Отметьте галочками, что вам подходит:
- [ ] **Fitch AI Drafting** — автогенерация сценария и UI по короткому описанию.

- [ ] **Fitch Multi-Target Output** — публикация одной сборки в web/desktop/terminal без ручной правки.
